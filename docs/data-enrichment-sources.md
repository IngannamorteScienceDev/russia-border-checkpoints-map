# Источники для обогащения данных КПП

Цель проекта - показывать интересную карточку КПП, но не выдавать непроверенные сведения за факт. Поэтому источники делятся на уровни доверия.

## Уровень 1. Основной операционный источник

**ФГКУ Росгранстрой, публичный слой карты**

- URL: `https://rosgranstroy.ru/api/map_data`
- Что берем: координаты, адрес, тип КПП, статус, режим работы, субъект РФ, сопредельную страну, сопредельный пункт, дату обновления.
- Как используем: это базовый machine-readable источник пайплайна.
- Ограничение: API не заменяет правовую сверку и может не объяснять, почему статус или инфраструктура изменились.

## Уровень 2. Официальная справочная сверка

**Минтранс России, раздел "Пункты пропуска через государственную границу Российской Федерации"**

- URL: `https://mintrans.gov.ru/activities/168`
- Что берем: контрольный перечень, классификацию, правовые документы и официальные публикации по пунктам пропуска.
- Как используем: сверяем наличие КПП, название, тип, субъект и правовой статус.
- Ограничение: часть материалов публикуется PDF/документами, поэтому потребуется полуавтоматический парсинг и ручная проверка спорных записей.

## Уровень 3. События и инфраструктура

**Новости Росгранстроя и Минтранса**

- Росгранстрой: `https://www.rosgranstroy.ru/press-center/news`
- Минтранс: `https://mintrans.gov.ru/press-center/news`
- Что берем: реконструкции, пропускную способность, сроки работ, электронную очередь, временные ограничения, открытие/закрытие объектов.
- Как используем: формируем короткие "инсайты" в паспорте КПП с датой, источником и уровнем уверенности.
- Ограничение: новости не должны перезаписывать базовый статус без подтверждения в официальном перечне или API.

## Уровень 4. Геоконтекст

**OSM/геокодинг/картографические слои**

- Что берем: ближайший населенный пункт, дороги, подъезды, расстояние до регионального центра, контекст местности.
- Как используем: только как пользовательский контекст, не как официальный источник статуса.
- Ограничение: такие сведения нужно помечать как "картографический контекст", а не как официальные данные.

## Правило для карточки КПП

Каждый новый факт должен иметь:

- `sourceUrl` - откуда взят факт;
- `sourceTitle` - человекочитаемое название источника;
- `sourceDate` - дата публикации или получения;
- `confidence` - `high`, `medium` или `low`;
- `kind` - `official`, `news`, `map_context` или `manual_review`;
- `summary` - короткая формулировка без пересказа длинных документов.

## Практический план

1. Оставляем Росгранстрой API как основной источник данных карты.
2. Добавляем source-audit в паспорт КПП, чтобы пользователь видел происхождение записи.
3. Затем строим отдельный enrichment-файл с фактами из Минтранса и Росгранстроя.
4. Для спорных записей создаем очередь ручной проверки, а не показываем сомнительные факты как подтвержденные.
